# BOSS直聘求职外挂客户端 - 需求规格说明书

## 一、项目概述

| 项目 | 说明 |
|------|------|
| **项目名称** | BOSS直聘智能求职助手 |
| **产品定位** | 个人使用的本地桌面端求职自动化工具 |
| **核心价值** | AI驱动 + 自动化操作，解放重复性求职沟通工作 |
| **部署方式** | 纯本地运行，无需远程服务器 |

## 二、用户角色与使用场景

### 目标用户
- 正在积极找工作的求职者
- 需要大量投递/沟通岗位的求职者
- 希望提高求职效率的个人用户

### 核心使用流程

#### 当前人工流程（待自动化替代）
```
① 完善简历 → ② 手动筛选岗位 → ③ 设置通用打招呼模板(所有岗位共用) → ④ 发送简历截图 → ⑤ 等对方回复 → ⑥ 回复后手动去豆包搜话术
```

#### 自动化目标流程（基于Browser Use浏览器自动化）
```
① 上传简历(AI解析→生成用户画像)
    ↓
② 启动自动求职模式（Browser Use打开BOSS直聘网页版）
    ↓
③ [循环] Browser Use提取当前页面DOM → AI理解岗位信息(JD/薪资/要求) → 判断是否符合标准
         → 符合则AI生成**针对该岗位的个性化打招呼语** → Browser Use自动点击"立即沟通"并输入发送
    ↓
④ [监听] Browser Use检测消息列表新回复标记 → **调用豆包AI生成回复话术** → 用户确认后发送 / 或全自动发送
```

> **技术升级说明**：采用 **Browser Use**（AI浏览器Agent框架）替代原"截图+CV视觉识别+pyautogui模拟"方案。
> 核心变化：从**像素级截图OCR**升级为**DOM级页面理解**，操作更精确、反检测更强、开发效率更高。

#### 核心痛点 vs 解决方案

| # | 人工痛点 | 自动化解决方案 | 价值 |
|---|----------|---------------|------|
| 1 | **手动筛选岗位**，逐个看耗时 | Browser Use自动提取页面DOM + AI理解岗位信息，按标准自动过滤 | 节省大量浏览时间 |
| 2 | **打招呼模板通用**，所有岗位用同一段话，没有针对性 | AI根据每个岗位的具体JD要求，**实时生成个性化打招呼语**（突出匹配的技能/经验） | 提高回复率 |
| 3 | **收到回复后要手动去豆包搜话术**，来回切换 | 检测到新回复后，**直接在工具内调用豆包AI生成回复建议**，一键复制或自动发送 | 响应速度提升10倍 |
| 4 | **重复性操作多**：翻页、点击、输入、发送 | **Browser Use全流程自动化**（DOM级精准操作+内置反检测），设置好标准后**挂机自动运行** | 解放双手 |

#### 2. 求职标准配置模块 `F002`

| F002-02 | 薪资范围 | 设置最低期望薪资和最高期望薪资 | P0 |

#### 3. 岗位信息识别模块 `F003`（基于Browser Use DOM提取）
| 编号 | 功能点 | 描述 | 优先级 |
|------|--------|------|--------|
| F003-01 | **Browser Use页面DOM提取** | **通过Browser Use自动提取BOSS直聘网页版当前页面的DOM结构，将交互元素（岗位卡片、按钮、链接等）转化为LLM可理解的结构化文本** | **P0 核心** |
| F003-02 | **AI理解岗位信息** | **将DOM结构化文本喂给LLM（豆包），让AI直接从DOM中理解并提取：岗位名称、公司名、薪资范围、岗位要求(JD)、工作经验要求、学历要求等**。**无需传统OCR，准确率更高** | **P0 核心** |
| F003-03 | 岗位信息结构化 | 将AI提取结果清洗为标准JSON数据（JobInfo模型） | P0 |
| F003-04 | **元素智能定位** | **Browser Use通过AI自然语言描述定位UI元素（如"点击立即沟通按钮"、"找到下一页按钮"），无需硬编码CSS选择器或屏幕坐标** | **P0** |
| F003-05 | 页面状态感知 | 检测当前处于哪个页面（搜索列表/岗位详情/聊天窗口/消息列表），据此切换操作策略 | P0 |
| F003-06 | 动态内容等待 | 自动处理SPA页面动态加载（BOSS直聘是React SPA），等待元素渲染完成后再操作 | P0 |

#### 4. AI匹配与话术生成模块 `F004`（核心差异化能力）
| 编号 | 功能点 | 描述 | 优先级 |

| F004-02 | 匹配度阈值过滤 | 低于设定阈值的岗位自动跳过（默认60分） | P0 |
| F004-03 | **个性化打招呼语生成** | **AI根据该岗位的具体JD要求+用户画像，实时生成针对性打招呼消息**（如：岗位要Python就突出Django经验，岗位要管理就突出团队Leader经历）。**每条不同，替代原来所有岗位共用一个模板的做法** | **P0 核心** |
| F004-04 | 话术风格选择 | 内置多种风格：正式专业/亲切自然/突出亮点/提问引导，用户可切换偏好 | P0 |

## 四、非功能性需求

| 类别 | 需求 | 说明 |
|------|------|------|
| **性能** | DOM提取→AI理解→决策→操作 全链路延迟 < 8秒 | Browser Use的DOM提取比截图OCR更快，单个岗位处理周期更短 |
| **可靠性** | 异常自动暂停 | 遇到验证码、弹窗、网络异常时Browser Use自动检测并暂停，通知用户 |
| **安全性** | API Key本地存储 | 火山引擎API Key加密存储在本地，不上传任何服务器 |
| **隐私** | 数据完全本地 | 所有数据（简历、画像、记录）仅存储在本地SQLite |
| **易用性** | 开箱即用 | 双击启动 → 配置API Key → 上传简历 → 开始使用 |
| **风控对抗** | Browser Use内置反检测 | undetectable browser模式 + 随机延迟 + 人类打字节奏模拟 + 操作频率控制 |
| **平台兼容** | BOSS直聘网页版 | 基于Chromium浏览器访问BOSS直聘网页版（www.zhipin.com），暂不支持客户端EXE |

## 五、技术选型确认

| 技术层 | 选型 | 版本/说明 |
|--------|------|-----------|
| **开发语言** | Python | 3.11+ |
| **GUI框架** | **Gradio** | Phase 1 快速原型，内置文件上传/聊天/文本组件，异步友好 |
| **打包方案** | **PyInstaller + 首次运行自动下载Chromium** | exe约50MB，首次运行自动playwright install chromium（需网络） |
| **AI Agent框架** | **PydanticAI** | LLM调用 + 结构化输出 + 话术生成 + 匹配评分 |
| **AI模型** | 火山引擎(豆包) | 大语言模型（兼容OpenAI协议，豆包-pro-32k/豆包-lite） |
| **浏览器自动化** | **Browser Use** | AI浏览器Agent，DOM提取+智能操作+内置反检测（MIT开源） |
| **浏览器内核** | Chromium (Playwright) | Browser Use底层基于Playwright驱动Chromium |
| **数据库** | SQLite | 本地轻量存储（用户画像、求职记录、对话历史） |
| **简历解析** | 火山引擎多模态模型(豆包-vision) | 简历图片/PDF内容结构化提取 |
| **配置管理** | YAML / TOML | 求职标准、API密钥、操作参数 |

## 七、风险与注意事项

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| **BOSS直聘风控检测** | 账号可能被限制/封禁 | Browser Use内置undetectable反检测 + 随机延迟(3-8s) + 限制每日数量(≤50) + 异常自动暂停 |
| **DOM结构变化** | BOSS前端改版导致DOM提取失效 | Browser Use的AI元素定位基于语义理解而非硬编码选择器，抗改版能力较强；保留CSS选择器降级方案 |
| **登录态过期** | 浏览器Cookie失效导致需要重新登录 | 会话管理模块检测登录态，过期时暂停并提示用户手动登录 |
| **API调用成本** | 频繁调用火山引擎产生费用 | 岗位信息本地缓存（同一岗位不重复解析）；轻量任务用豆包-lite节省成本；话术生成用pro保证质量 |
| **法律合规性** | 自动化操作可能违反BOSS平台用户协议 | 仅个人使用、控制频率、免责声明；建议在设置中加入"使用即知悉风险"确认 |
| **Browser Use依赖** | 上游库更新可能引入breaking change | 锁定版本号(pip freeze)；关注browser-use官方changelog |
| **网络波动** | 网络不稳定导致操作中断 | 操作重试机制(最多3次) + 断点续传（记录已处理到哪个岗位） |

### 关键依赖版本锁定策略

| 依赖包 | 用途 | 建议初始版本 |

| browser-use | 浏览器自动化Agent | latest stable |
| pydantic-ai | AI Agent框架 | >=0.0.40 |

| gradio | GUI界面框架 | >=5.0 |

### 客户端打包方案

**选型：PyInstaller + 首次运行自动下载Chromium**

```
打包产物：
dist/
├── BOSS求职助手.exe              # 主程序 (~50MB, 不含Chromium)
├── browsers/                     # 首次运行后自动下载
│   └── chromium-xxx/
├── config/
│   ├── settings.yaml             # 求职标准配置
│   └── api_config.yaml           # API密钥（首次填写）
├── data/
│   ├── job_assistant.db          # SQLite数据库
│   └── profiles/                 # 用户画像JSON
└── logs/                         # 运行日志
```

**核心要点：**
- `pyinstaller -F --noconsole` 打包为单文件无控制台exe
- Chromium 不内嵌到exe中（节省~250MB体积）
- 首次运行检测Chromium是否存在，不存在则自动调用 `playwright install chromium`
- 显示下载进度条，下载完成后正常启动
- UPX压缩减小体积
